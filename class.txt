### github 활용 : 포트폴리오, 협업, 포트폴리오 검증

mkdir : make directory

ls : list

cd : change directory

rm : remove

- ‘#’ : 제목 (글자 크게/개수에 따라 글자 크기 상이)
- 양옆에 ‘**’ : 글자 굵게
- 양옆에 ‘*’ : 글자 기울기
- 양옆에 ‘~~’ : 취소선

이유 : 제목은 1개다. 내 모든 HTML 페이지에 H1태그.기계가 내 글을 잘 이해함

git : 분산 버전 관리 시스템

- Working Directory : 실제 작업 폴더
- Staging Area : 임시 공간 (눈에 보이지 않음)
- Repository : 저장 공간 (모든 버전+변경 이력 기록), 최종 push 공간

1. **add** : (Working Directory → Staging Area)
2. **commit** : 변경 파일을 저장하는 행위 / 버전 이름, 내용을 적어둠
3. **push** : repository 로 보내는 행위

commit 메세지가 없으면 push를 해도 github repository에 안 보인다.
